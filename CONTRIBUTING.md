## How to Contribute

### 1. Fork the repository
Click the **Fork** button on GitHub and clone your fork locally.

### 2. Create a branch
Use a descriptive branch name with convention below

- feature/xxx-xxx-xxx
- fix/xxx-xxx
- docs/xxx-xxx


### 3. Make your changes
- Keep changes focused and small
- Add or update tests where relevant
- Update documentation if behavior changes

### 4. Run checks
Before submitting:
- Ensure tests pass
- Ensure the project runs locally
- Ensure no secrets or API keys are committed

### 5. Open a Pull Request
- Describe **what** you changed and **why**
- Link any related issues
- Add screenshots or examples if helpful

## Development Guidelines

- Follow existing code style
- Prefer clear, readable code over clever code
- Keep public APIs backwards compatible when possible

## Reporting Issues

If you find a bug:
- Check existing issues first
- Include steps to reproduce
- Include logs or error messages if possible

## Feature Requests

Feature ideas are welcome!
- Explain the use case
- Explain why it benefits the project
- Keep scope reasonable

## Code of Conduct

By participating, you agree to follow the Code of Conduct.
